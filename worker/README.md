# Live NEEP lookup without a proxy for end users

NEEP's API blocks browser cross-origin requests, so the PWA can't call it
directly. The robust fix is a **tiny Cloudflare Worker you deploy once** — after
that, the app calls your worker, which fetches NEEP server-side and returns the
data with open CORS headers. **End users never configure anything.**

## Deploy (free, ~1 minute)

```bash
npm i -g wrangler
# authenticate with a Cloudflare API token (CF_API_TOKEN) or global key:
export CLOUDFLARE_API_TOKEN="cfat_xxx"   # OR:
export CLOUDFLARE_API_KEY="cfk_xxx"; export CLOUDFLARE_EMAIL="you@example.com"
mkdir dualfuel-proxy && cd dualfuel-proxy
# copy proxy.js + wrangler.toml here, then:
wrangler deploy
```

> **First-time workers.dev subdomain:** Cloudflare has no API to *create* a
> `*.workers.dev` subdomain — it is only created when the Workers & Pages
> dashboard is opened, or via a `PUT` using your **Global API Key**:
> ```bash
> curl -X PUT "https://api.cloudflare.com/client/v4/accounts/$ACCOUNT_ID/workers/subdomain" \
>   -H "X-Auth-Email: $CF_EMAIL" -H "X-Auth-Key: $CF_KEY" \
>   -H "Content-Type: application/json" --data '{"subdomain":"dualfuel-balance"}'
> ```
> (A scoped API token cannot create it — you get `10405`/`405`.) Do this once,
> then `wrangler deploy` publishes to `https://dualfuel-proxy.<subdomain>.workers.dev/`.

## Point the app at it

This project's `DEFAULT_PROXY` already points at the deployed worker:

```js
var DEFAULT_PROXY = "https://dualfuel-proxy.dualfuel-balance.workers.dev/";
```

(or just enter it in **Settings ⚙** inside the app — no code edit needed).

Now model searches hit NEEP live with full coverage, and no end user touches a
proxy. If the worker is ever unreachable, the app automatically falls back to
the bundled catalog (`data/heatpumps.js`) and manual entry.

## Alternative: host an expanded catalog on a CDN

`data/heatpumps.js` (generated by `scripts/fetch_catalog.py`) already bundles
thousands of real models and works fully offline. If you want even broader
coverage with zero backend, commit that file to a GitHub repo and load it via
jsDelivr (`https://cdn.jsdelivr.net/gh/<user>/<repo>@<tag>/data/heatpumps.js`),
which sends `Access-Control-Allow-Origin: *`. The app can fetch it directly —
no proxy, no Worker.
